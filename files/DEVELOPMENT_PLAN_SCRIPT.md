# 개발 계획 발표 대본

---

## 슬라이드 순서

```
1. 요구사항 정의서   →   "무엇을 해야 하는가?"
2. 기능 정의서       →   "어떤 기능이 필요한가?"
3. WBS              →   "어떻게 나눠서 할 것인가?"
4. 간트 차트         →   "언제까지 할 것인가?"
5. 요구사항 추적     →   "제대로 다 했는가?"
```

---

## 슬라이드 1: 요구사항 정의서

### 발표 멘트

"먼저 **요구사항 정의서**입니다.

저희 프로젝트의 핵심 요구사항은 크게 6가지 영역으로 분류됩니다.

**개요** 영역에서는 프로젝트 목적과 시스템 구조를 정의했습니다.
저희는 사용자가 입력한 이미지와 텍스트를 바탕으로 **AI가 자동으로 잡지 레이아웃을 생성**하는 시스템을 만드는 것이 목표입니다.

**기능** 영역에서는 사용자 입력, Safety Agent, Vision Agent, RAG 시스템, MCP Server, HTML 생성 등 총 6개의 핵심 기능을 정의했습니다.

**성능** 영역에서는 생성 속도 25초 이내, 검색 응답 0.5초 이내 등 성능 목표를 설정했습니다.

**품질** 영역에서는 품질 점수 90점 이상, 겹침률 2% 미만 등 품질 기준을 정의했습니다.

**데이터** 영역에서는 122개 레이아웃 DB와 104개 템플릿을 구축하는 것을 목표로 했습니다.

마지막으로 **인프라** 영역에서는 FastAPI, Gemini API, Voyage API 등 기술 스택을 정의했습니다.

총 **16개의 요구사항**을 수립했으며, 모두 '수행' 상태로 완료되었습니다."

---

## 슬라이드 2: 기능 정의서

### 발표 멘트

"다음은 **기능 정의서**입니다.

요구사항을 충족하기 위해 어떤 기능들이 필요한지 상세하게 정의했습니다.

저희 시스템은 크게 **4가지 구분**으로 나뉩니다.

**USER** 영역은 일반 사용자가 사용하는 기능입니다.
콘텐츠 입력, 레이아웃 생성, 결과 미리보기, 다운로드 등 9개 기능이 포함됩니다.

**Guest** 영역은 비회원을 위한 기능입니다.
서비스 소개, 데모 체험 등 3개 기능이 포함됩니다.

**SYSTEM** 영역이 가장 핵심입니다.
Intent Router부터 Safety, Vision, Planner, Director, Editor, Publisher, Critique, MCP Server, RAG System, Image Validator까지 총 **40개 이상의 세부 기능**이 포함됩니다.

**ADMIN** 영역은 관리자용 기능입니다.
데이터 관리, 템플릿 관리, 모니터링 등 5개 기능이 포함됩니다.

특히 각 기능별로 **입력값/출력값**, **예외 처리**, **화면 설계**까지 상세하게 정의하여 개발팀이 명확하게 구현할 수 있도록 했습니다."

---

## 슬라이드 3: WBS (Work Breakdown Structure)

### 발표 멘트

"다음은 **WBS**, 작업 분해 구조입니다.

정의된 기능들을 **5개의 Phase**로 나누어 작업을 분해했습니다.

**Phase 0**은 기획 단계입니다. 12월 24일부터 30일까지 주제 선정, 노드 구성 설계, 파이프라인 확정을 진행했습니다.

**Phase 1**은 기반 구축 단계입니다. 12월 31일부터 1월 5일까지 아키텍처 설계, Vision Agent 개발, Core 파이프라인 개발, 렌더링 개발을 진행했습니다.

**Phase 2**는 지능화 단계입니다. 1월 6일부터 9일까지 시스템 안정화, 성능 최적화, RAG 도입, Safety 개발을 진행했습니다.

**Phase 3**은 확장 단계입니다. 1월 10일부터 12일까지 Agent 리팩토링, 데이터 인프라 구축, Embedding 전략 수립을 진행했습니다.

**Phase 4**는 완성 단계입니다. 1월 13일부터 15일까지 MCP 최종화, 품질 확보, 문서화를 진행했습니다.

총 **4명의 팀원**이 각자의 영역을 담당했습니다.
- **유신**: 아키텍처, RAG 파이프라인, 데이터 인프라
- **규리**: Vision Agent, Embedding 전략, 렌더링 품질
- **명래**: Core 파이프라인, MCP Server, 시스템 재설계
- **승민**: Safety Agent, 프롬프트 엔지니어링, 이미지 최적화"

---

## 슬라이드 4: 간트 차트

### 발표 멘트

"다음은 **간트 차트**입니다.

WBS를 시간축으로 시각화한 것입니다.

**12월 24일**부터 **1월 15일**까지 약 **3주간**의 개발 일정입니다.

첫 주는 **기획과 기반 구축**에 집중했습니다.

둘째 주는 **지능화와 확장**에 집중했습니다. 특히 이 시기에 RAG 시스템과 Agent 리팩토링이 병렬로 진행되었습니다.

셋째 주는 **MCP 완성과 최종 품질 확보**에 집중했습니다.

핵심 마일스톤 5개를 설정했습니다.

1월 5일: 9개 노드 파이프라인 확정
1월 9일: Placeholder 치환으로 10배 속도 향상
1월 11일: A2A 마이그레이션 완료, 1,960줄 정비
1월 12일: RAG 시스템 완성, 122개 레이아웃 DB
1월 15일: MCP Server 완성, 최종 배포"

## 슬라이드 5: 요구사항 추적 매트릭스

### 발표 멘트

"마지막으로 **요구사항 추적 매트릭스**입니다.

이 문서는 모든 요구사항이 **설계, 구현, 테스트**와 빠짐없이 연결되어 있는지 추적하는 문서입니다.

예를 들어, **RU06 RAG 시스템** 요구사항을 보면,
- **설계**: RAG 아키텍처 설계
- **구현**: `rag_voyage.py`, Voyage 3.5 임베딩, ChromaDB, 4-Stage Fallback
- **테스트**: 검색 정확도 테스트, Fallback 테스트

이렇게 요구사항부터 테스트까지 **일관된 흐름**으로 연결되어 있습니다.

총 16개 요구사항 모두 **설계 완료, 구현 완료, 테스트 완료** 상태입니다.

이를 통해 저희는 **요구사항 누락 없이** 프로젝트를 완수했음을 증명할 수 있습니다."
